# HH.ru Parser

Парсер вакансий с сайта hh.ru с использованием публичного API. Консольное приложение на C++ для сбора и анализа вакансий по ключевым словам.

## Цель разработки

Создать приложение, демонстрирующее:

- Работу с HTTP-запросами
- Парсинг JSON-данных
- Принципы ООП:
  - Инкапсуляция
  - Наследование
  - Полиморфизм
  - Абстракция

## Общая архитектура

Проект построен на иерархии классов с использованием абстрактного базового класса `Parser`. Все данные управляются контейнером `ParserManager`.

### Основные компоненты:

**Иерархия парсеров**
- `Parser` (абстрактный класс)
- Производные классы:
  - `HHParser` (парсер hh.ru)

**Модель данных**
- `Vacancy` — класс для хранения информации о вакансии:
  - ID, название, компания, зарплата (from/to), описание, ссылка, город

**Менеджер парсинга**
- `ParserManager` — управляет:
  - Сбором данных
  - Координацией работы парсеров
  - Сохранением результатов

**Сохранение данных**
- Экспорт в CSV-формат

## Основной функционал

### 1. Управление поиском
- Ввод ключевого слова для поиска
- Фиксированное количество страниц (3 страницы по 20 вакансий = 60 вакансий)

### 2. Парсинг данных
- Отправка GET-запросов к API hh.ru
- Обработка JSON-ответов
- Извлечение полей вакансий
- Rate limiting (1 запрос в секунду)

### 3. Отчеты и статистика
- Вывод результатов в консоль
- Сохранение в CSV
- Подсчет статистики:
  - Общее количество вакансий
  - Количество вакансий с указанием зарплаты
  - Средняя зарплата
  - Самый популярный город

## Требования к реализации

### Обязательные
- [x] Абстрактный класс `Parser` с чисто виртуальным методом `parse()`
- [x] Композиция: `ParserManager` содержит коллекцию парсеров
- [x] Динамическое создание объектов
- [x] Полиморфный вызов методов парсинга
- [x] Работа с JSON через RapidJSON
- [x] HTTP-запросы через CPR

### Дополнительные
- [x] Консольный интерфейс с меню
- [x] Обработка ошибок (таймауты, недоступность API)
- [x] Rate limiting для соблюдения правил API
